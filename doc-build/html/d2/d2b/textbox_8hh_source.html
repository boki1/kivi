215 /* An utility function that can be used to create a tree graph rendering from a structure.
216  *
217  * Parameters:
218  * e: The element that will be rendered.
219  * Possibly some user-defined type that represents a node in a tree structure.
220  * maxwidth: The maximum width of the resulting box in characters.
221  * create_atom: A functor of type std::string(const ParamType&).
222  * It renders the given element into an 1D string.
223  * Note that the string must not contain multibyte characters,
224  * because size() will be used to determine its width in columns.
225  * count_children: A functor of type std::pair<ForwardIterator,ForwardIterator>(const ParamType&).
226  * It returns a pair of iterators representing the range of children
227  * for the given element.
228  * create_tree_graph will call itself recursively for each element in this range.
229  * oneliner_test: A functor of type bool(const ParamType&).
230  * If the result is true, enables simplified horizontal topology.
231  * simple_test: A functor of type bool(const ParamType&).
232  * If the result is true, enables very simplified horizontal topology.
233  * separate1st_test: A functor of type bool(const ParamType&).
234  * If the result is true, create_tree_graph() will always render
235  * the first child alone on a separate line, but the rest of them
236  * may get rendered horizontally.
237  *
238  * Topology types:
239  *
240  * Vertical:
241  *
242  * element
243  * ├─child1
244  * ├─child2
245  * └─child3
246  *
247  * Horizontal:
248  *
249  * element
250  * └─┬─────────┬─────────┐
251  * child1 child2 child3
252  *
253  * Simplified horizontal:
254  *
255  * element──┬───────┬───────┐
256  * child1 child2 child3
257  *
258  * Very simplified horizontal:
259  *
260  * element──child1
261  *
262  * The vertical and horizontal topologies are automatically chosen
263  * depending on the situation compared to the maxwidth parameter,
264  * and according to the constraint given by separate1st_test().
265  *
266  * Simplified topology will be used if oneliner_test() returns true,
267  * separate1st_test() returns false,
268  * all children fit on one line,
269  * and very simplified topology is not used.
270  *
271  * Very simplified topology will be used if oneliner_test() returns true,
272  * separate1st_test() returns false,
273  * simple_test() returns true,
274  * there is only 1 child,
275  * and it fits on one line.
